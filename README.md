# WebRTC 기반 포인트 클라우드 스트리밍 최적화 실험

대용량 3D 포인트 클라우드 데이터의 실시간 스트리밍 한계를 극복하고 최적의 사용자 경험을 제공하기 위해 설계되었다. 방대한 3D 데이터를 네트워크를 통해 한 번에 전송하는 것은 막대한 부하를 일으키기 때문에, 다음과 같은 기술적 접근을 통해 실험의 타당성을 검증하였다.

* **네트워크 부하 감소를 위한 타일링 기법**: 3D 데이터의 방대한 용량 문제를 해결하기 위해 전체 데이터를 전송하는 대신 공간을 타일 단위로 분할하였다. 현재 사용자가 바라보는 시야 영역의 데이터만 선택적으로 불러와 불필요한 데이터 전송을 막았다.

* **사용자 경험을 위한 혼합 화질 로딩**: 시야에 노출된 영역만 불러올 경우 시선이 이동할 때 화면이 비어 보이는 시각적 단절이 발생한다. 이를 방지하고자 시야 밖의 영역에는 저화질 타일을 미리 로딩하였다. 이는 타일을 처리하는 지연시간이 미세하게 증가하더라도, 끊김 없이 자연스러운 스트리밍 환경을 제공하여 전반적인 사용자 경험을 향상시키기 위한 필수적인 구조이다.

* **실시간 통신 및 압축 효율 극대화**: 실시간 스트리밍을 구현하고 요구되는 대역폭을 효과적으로 줄이기 위해 WebRTC 프로토콜을 활용하였다. 또한, 무거운 원본 PLY 파일이 가진 물리적 용량의 한계를 극복하기 위해 고효율 압축 방식인 Draco 인코딩을 적용하여 전송 데이터를 획기적으로 줄였다.

## 2. 데이터 전처리 (Data Preprocessing)
원본 포인트 클라우드 PLY 파일을 스트리밍에 최적화된 DRC 파일로 사전 가공하는 파이프라인이다. 데이터 전처리 폴더의 파이썬 스크립트를 통해 다음 순서로 실행된다.\
1. **3D 타일링 (`tiling.py`)**: 오브젝트의 바운딩 박스를 기준으로 설정된 타일 크기에 따라 전체 데이터를 분할하여 개별 타일 파일을 생성한다.
2. **다운샘플링 (`downsampling.py`)**: Open3D 라이브러리에서 제공하는 Voxel Size를 기반으로 포인트 수를 줄여 원본 데이터의 크기를 경량화한다.
3. **Draco 인코딩 (`PlytoDrc.py`)**: 파이썬 환경에 구축한 Draco 인코더를 통해 타일 데이터를 압축한다. 스트리밍 환경과 화질 요구사항에 맞추어 압축 레벨과 양자화 비트 값을 조절하여 고효율의 DRC 파일을 생성한다.
4. **XML (`xml_generator.py`)**: 분할된 개별 타일들이 렌더링 시 하나의 온전한 프레임처럼 보이도록 각 타일의 중심 좌표를 계산한다. 이 위치 정보와 각 타일의 화질 정보를 모두 담은 XML 파일을 생성하여 전체 스트리밍 구조를 관리한다.

## 3. 유니티 스트리밍 시스템 (Unity System)
단일 로컬 PC 환경에서 송신부와 수신부 두 개의 독립된 인스턴스를 통해 실시간 스트리밍을 구현한다. WebRTC를 활용하여 전송에 필요한 대역폭을 효과적으로 절감하는 데 집중하였다.

### 📡송신부
* **맞춤형 데이터 전송**: 전처리 과정을 거친 타일 사이즈와 파일 포맷 규격에 맞추어 WebRTC Data Channel을 통해 수신부로 데이터를 전송한다.

### 📺수신부
* **FoV 기반 타일 로딩**: 수신부 카메라의 FoV와 사전에 생성된 XML 메타데이터를 기반으로, 현재 시야에 맞게 필요한 타일들을 선택적으로 불러온다.
* **실시간 복원 및 모니터링**: 수신된 데이터를 실시간으로 디코딩하여 3D 포인트 클라우드 형태로 렌더링한다. 동시에 전체 스트리밍 파이프라인에서 소모되는 대역폭과 타일을 불러오는 지연시간을 측정하여 실시간으로 출력한다.
